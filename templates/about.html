About This Intelligent Job Matching Backend System

This page is a live documentation page for the entire backend. It explains the project structure, models, relationships, matching logic, routes, admin panel, UI pages, access control, and future scope.

1. Project Overview

The project is a template-based Django job matching system. It supports candidate registration, recruiter registration, company-managed hiring, job posting, skill-based matching, applications, recruiter ranking of candidates, and candidate ranking of jobs.

The high-level idea is simple: candidates store their skills, recruiters create jobs with required skills, and the backend computes scores to match the right candidate with the right job.

2. System Architecture

  • accounts - custom user model, login, register, logout, and role-based decorators.
  • candidates - candidate profile, candidate skills, and candidate skill management page.
  • companies - company model and recruiter profile linked to a company.
  • jobs - job creation, job listing, recruiter dashboard, applicant views, and matching views.
  • core - shared Skill model used across jobs and candidates.
  • applications - job application workflow and applied jobs page.

3. Models Breakdown

Model Fields Relationships Purpose

4. Database Relationships

  • OneToOne (User ↔ Profile) - each user is exactly one candidate profile or one recruiter profile, which keeps the role identity clean and simple.
  • ForeignKey (Job → Company) - a job belongs to one company so company ownership stays clear.
  • ForeignKey (Job → RecruiterProfile) - a job also stores who created it, so recruiter actions are traceable.
  • Through-style mapping models (CandidateSkill, JobSkill) - these are used to store many skills against candidates/jobs plus extra data like proficiency, importance, and required level.

5. Core Functionalities

  • Matching Engine - compares candidate skills against job skills and gives a score.
  • Job Application System - candidates can apply to jobs and the application is stored with status and timestamp.
  • Candidate → Job recommendation - the system shows jobs ranked by score for a logged-in candidate.
  • Recruiter → Candidate ranking - the system shows top candidates for a given job based on matching score.

7. Matching Algorithm

The matching logic is implemented in jobs/services/matching.py. It uses candidate skills and job required skills.

Part Explanation
Level mapping beginner = 1, intermediate = 2, expert = 3
Importance weights nice_to_have = 1, preferred = 2, very_important = 3
Score formula level_ratio × importance_weight × 10
Level ratio candidate_level / required_level, capped at 1.0

This means a candidate with the right skill and a strong level gets a higher score. Jobs and candidates are then sorted by score.

9. Authentication & Decorators

  • login_required protects pages that should only be reachable by authenticated users.
  • unauthenticated_user redirects logged-in users away from login and register pages.
  • candidate_required makes sure only candidates can reach candidate-only actions.
  • recruiter_required makes sure only recruiters can reach recruiter-only actions.

10. Admin Panel

  • Registered models: User, CandidateProfile, CandidateSkill, Company, RecruiterProfile, Skill, Job, JobSkill, JobApplication.
  • Usage: admin panel is used to manage core data, add companies, inspect users, test the demo dataset, and update application statuses.
  • Why it matters: it gives a fast backend control center for demos and interviews.

11. Pages / UI

  • Home page - entry page with login/register buttons and app branding.
  • Matches page - candidate jobs ranked by matching score or recruiter top candidates page.
  • Applied jobs page - candidate’s application history and statuses.
  • Job listing page - public list of jobs, with match-aware ordering for logged-in candidates.

12. Future Improvements

  • Add filters for location, job type, salary, and match score.
  • Upgrade to ML-based ranking instead of only rule-based matching.
  • Add DRF APIs for mobile or frontend integrations.
  • Add edit/update skill management for candidates.
  • Add pagination and search across job listing and applicant lists.
